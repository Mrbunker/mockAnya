# Electron 桌面应用需求设计

## 一、完整需求

### 项目目标与范围

- 构建离线可用的 Electron 桌面应用，支持自动生成多种模拟文件
- 文件类型：图片类/视频类/文本类
- 支持参数配置、生成进度、文件落盘与摘要信息
- 优先采用成熟社区包与官方工具链，不重复造轮子s

### 文件类型与参数
已文件类型进行分类（如图片）
- 图片类
  - 格式：PNG/JPEG
  - 宽度/高度：用户自定义 16–4096
  - 背景色：默认纯黑，可选纯色/棋盘格
  - 文件大小：最大 1024MB
- 视频类
  - 格式：MP4
  - 时长：1s–10min
  - 分辨率：预设 360p/720p/1080p 与自定义（如 1280×720）
  - 帧率： 30/60/120
  - 内容：纯色视频（黑场）
  - 文件大小：自定义最大 1024MB
- 音频类
  - 格式：MP3
  - 时长：1s–10min
  - 文件大小：自定义最大 1024MB
- 文本类
  - 格式：TXT/CSV/JSON/PDF
  - 文件大小：自定义最大 1024MB
  - 写入重复内容
- 安装包
  - 格式：APK
  - 符合 android 的包名与版本号
  - 文件大小：自定义最大 1024MB
  

### 技术架构

- 前端：`Electron` `React + TypeScript`
- 脚手架 electron-vite
- 组件库 semideisgn,保持一致的 UI 风格与交互体验
- 使用 TailwindCSS 进行样式定制
- 依赖管理 pnpm

### ui 风格与页面设计

严格使用 [semideisgn] 默认主题设计风格，如：无渐变，无阴影；
首页使用 grid 卡片，每一种文件类型分别对应一个卡片；
点击卡片后进入相应表单，完成参数配置后，点击生成按钮，即可开始生成文件。
